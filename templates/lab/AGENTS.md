# Инструкция для ИИ-агента: шаблон лабораторной работы

Эта папка является шаблоном короткого отчета по лабораторной работе. Примеры
реальных отчетов имеют такую структуру:

1. Титульный лист.
2. `Цель работы: ...`
3. `Исходные данные (вариант N): ...`
4. `Ход работы: ...`
5. Рисунки, таблицы, листинги и пояснения по шагам.
6. `Вывод: ...`

## Главные правила

- Не меняй `vkr.cls`, `setup.tex` и `xltabular.sty` внутри созданного документа без отдельной просьбы.
- Не добавляй `\tableofcontents` и не используй `\section` в лабораторной работе. В этом классе `\section` начинает новую страницу, а лабораторные из примеров идут непрерывным текстом.
- Основной текст редактируй в `sections/01_goal.tex`, `sections/02_initial_data.tex`, `sections/03_work_progress.tex`, `sections/04_conclusion.tex`.
- Переменные титульного листа находятся в `main.tex`: номер лабораторной, вариант, дисциплина, тема, автор, группа, преподаватель.
- Пиши отчет в прошедшем времени или нейтральном стиле: "На рисунке 1 показано...", "В таблице 1 представлены...", "В ходе выполнения лабораторной работы...".
- Перед сборкой убедись, что все изображения, на которые есть ссылки, лежат в `images/`.

## Как вставлять рисунки

Используй helper из `lab-helpers.tex`:

```tex
\labimage{cache-settings.png}{Настройки кэш-памяти и ассемблированная программа}
```

Если нужен другой размер:

```tex
\labimage[0.75\textwidth]{step-01.png}{Выполнение команды IN}
```

Подпись должна быть содержательной. Не пиши просто "Скриншот".

## Как оформлять таблицы

Таблицы вставляй обычным LaTeX-окружением:

```tex
\begin{table}[H]
  \caption{Последовательность выполнения команд программы}
  \begin{tabular}{|c|c|c|c|}
    \hline
    Шаг & PC & Acc & Описание \\
    \hline
    0 & 000 & 000000 & Начальное состояние \\
    \hline
  \end{tabular}
\end{table}
```

## Как оформлять ход работы

В `03_work_progress.tex` сохраняй порядок из методички:

1. Короткое пояснение перед рисунком или таблицей.
2. Рисунок/таблица/листинг.
3. Пояснение результата.
4. Следующий шаг.

Пример формулировки:

```tex
На рисунке 1 показано состояние ЭВМ перед началом работы. Набор команд
варианта \Вариант введен и скомпилирован.

\labimage{initial-state.png}{Состояние ЭВМ перед началом работы}
```

## Как писать вывод

Вывод должен быть коротким и повторять цель через результат:

```tex
\labconclusion{в ходе выполнения лабораторной работы были изучены принципы
работы кэш-памяти процессора и исследован механизм обращения процессора к
памяти при выполнении программы.}
```

## Сборка

После правок вызови build API:

```http
POST /api/build
{
  "projectId": "<project-id>",
  "documentId": "<document-id>"
}
```

PDF должен сохраниться в `build/main.pdf`, лог сборки -- в `build/main.log`.
